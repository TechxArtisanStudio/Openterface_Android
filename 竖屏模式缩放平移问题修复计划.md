# 竖屏模式缩放平移问题修复计划

## 问题描述

### 问题 1：放大后显示范围太小
- **现象**：竖屏模式下双指捏合放大后，能看到的视频区域比预期小
- **用户描述**："放大之后的显示范围太小了"

### 问题 2：视角移动速度过快
- **现象**：放大后移动鼠标时，视角跟随移动的速度太快，无法精确控制
- **用户描述**："移动视角快了应该跟随鼠标的移动速度移动视角"
- **期望行为**：视角移动速度应该与鼠标移动速度成比例，而不是鼠标位置直接决定平移量

---

## 根本原因分析

### 问题 1：显示范围太小

**当前代码逻辑**：
```java
// CustomTouchListener.autoPanToKeepMouseCentered()
float maxNdcTranslate = scale - 1.0f;  // scale=2 时，maxTranslate=1.0 NDC
```

**数学分析**：
- 在 NDC 空间中，纹理顶点范围是 `[-1, 1]`
- 应用 `scale=2` 后，纹理范围变成 `[-2, 2]`
- 平移限制 `maxTranslate = scale - 1 = 1.0`
- 可见视口是 `[-1, 1]`

当 `tx = 1.0` 时：
- 纹理范围：`[-2+1, 2+1] = [-1, 3]`
- 可见范围：`[-1, 1]`
- 对应纹理区域：`[-1-1, 1-1]/2 = [-1, 0]`（只有纹理的左半部分）

当 `tx = -1.0` 时：
- 纹理范围：`[-2-1, 2-1] = [-3, 1]`
- 可见范围：`[-1, 1]`
- 对应纹理区域：`[-1+1, 1+1]/2 = [0, 1]`（只有纹理的右半部分）

**结论**：
- `scale=2` 时，只能看到纹理宽度的 1/2
- `scale=3` 时，只能看到纹理宽度的 1/3
- 这是正确的放大效果，但用户可能期望能看到更多区域

**可能的改进方向**：
1. 降低最大缩放倍数（当前 `MAX_SCALE = 3.0`）
2. 或者调整平移范围限制公式（但会牺牲边缘裁剪的正确性）

---

### 问题 2：视角移动速度过快

**当前代码逻辑**：
```java
// CustomTouchListener.autoPanToKeepMouseCentered()
float mouseNdcX = (mouseX - viewWidth / 2f) / (viewWidth / 2f);  // 鼠标 NDC 位置 [-1, 1]
float targetNdcX = mPortraitTranslateX - mouseNdcX * scale;       // 直接映射到平移量
```

**问题分析**：
1. 当前公式让**鼠标位置**直接决定**平移量**，而不是**鼠标移动速度**决定**视角移动速度**
2. 当鼠标从视口中心移动到边缘（`mouseNdcX` 从 0 变到 1）：
   - 平移增量：`deltaTx = 1.0 * scale`（例如 `scale=2` 时，增量是 2.0 NDC）
   - 但最大平移范围是 `scale - 1 = 1.0`
   - 所以**鼠标移动一个视口宽度就能达到最大平移范围**
3. 这导致视角移动速度过快，用户无法精确控制

**数学示例**：
- 假设 `scale=2`，`maxTranslate=1.0`
- 鼠标从中心移动到右边缘（`mouseNdcX` 从 0 到 1）
- 平移从 `tx=0` 变到 `tx = 0 - 1.0 * 2 = -2.0`
- 但被钳制到 `tx = -1.0`（最大值）
- **结果**：鼠标移动半个屏幕，视角就移到最右端

**根本原因**：
当前公式是错误的。应该让**鼠标移动增量**决定**视角移动增量**，而不是鼠标位置决定视角位置。

---

## 修复方案

### 方案 1：修复视角移动速度（问题 2）

**核心问题**：当前代码使用鼠标**绝对位置**计算视角**绝对位置**，应该改为使用鼠标**移动增量**计算视角**移动增量**。

**当前错误逻辑**（CustomTouchListener.java 第483-484行）：
```java
// 错误：用鼠标位置直接计算平移量
float targetNdcX = mPortraitTranslateX - mouseNdcX * scale;
float targetNdcY = mPortraitTranslateY - mouseNdcY * scale;
```

**正确逻辑**：
```java
// 正确：用鼠标移动增量计算平移增量
float deltaX = currentMouseNdcX - lastMouseNdcX;
float deltaY = currentMouseNdcY - lastMouseNdcY;
float translateDeltaX = -deltaX * sensitivity;
float translateDeltaY = -deltaY * sensitivity;
mPortraitTranslateX += translateDeltaX;
mPortraitTranslateY += translateDeltaY;
```

**修改文件**：`CustomTouchListener.java`

**修改内容**：

1. 添加字段：
```java
private float mLastMouseNdcX = 0f;
private float mLastMouseNdcY = 0f;
private boolean mHasLastMousePosition = false;
private static final float PAN_SENSITIVITY = 0.5f;  // 灵敏度参数
```

2. 修改 `autoPanToKeepMouseCentered()` 方法：
```java
private void autoPanToKeepMouseCentered(float mouseX, float mouseY) {
    if (activity == null || activity.mCameraHelper == null) {
        return;
    }

    int viewWidth = activity.mBinding.viewMainPreview.getWidth();
    int viewHeight = activity.mBinding.viewMainPreview.getHeight();

    if (viewWidth <= 0 || viewHeight <= 0) return;

    float scale = mPortraitZoomScale;

    // 计算当前鼠标 NDC 位置（相对于视口中心）
    float mouseNdcX = (mouseX - viewWidth / 2f) / (viewWidth / 2f);
    float mouseNdcY = (mouseY - viewHeight / 2f) / (viewHeight / 2f);

    // 首次调用时，只记录位置，不移动视角
    if (!mHasLastMousePosition) {
        mLastMouseNdcX = mouseNdcX;
        mLastMouseNdcY = mouseNdcY;
        mHasLastMousePosition = true;
        return;
    }

    // 计算鼠标移动增量
    float deltaMouseNdcX = mouseNdcX - mLastMouseNdcX;
    float deltaMouseNdcY = mouseNdcY - mLastMouseNdcY;

    // 将鼠标增量映射到视角平移增量（带灵敏度）
    float deltaTranslateX = -deltaMouseNdcX * PAN_SENSITIVITY;
    float deltaTranslateY = -deltaMouseNdcY * PAN_SENSITIVITY;

    // 累加到当前平移量
    float targetNdcX = mPortraitTranslateX + deltaTranslateX;
    float targetNdcY = mPortraitTranslateY + deltaTranslateY;

    // 钳制到有效范围
    float maxNdcTranslate = scale - 1.0f;
    targetNdcX = Math.max(-maxNdcTranslate, Math.min(maxNdcTranslate, targetNdcX));
    targetNdcY = Math.max(-maxNdcTranslate, Math.min(maxNdcTranslate, targetNdcY));

    // 更新平移量
    if (Math.abs(targetNdcX - mPortraitTranslateX) > 0.001f ||
        Math.abs(targetNdcY - mPortraitTranslateY) > 0.001f) {
        mPortraitTranslateX = targetNdcX;
        mPortraitTranslateY = targetNdcY;

        // 应用到 GL
        activity.mCameraHelper.setZoomScale(mPortraitZoomScale, mPortraitTranslateX, mPortraitTranslateY);

        // 更新 PiP 指示器
        if (portraitPiPManager != null && portraitPiPManager.isPiPVisible()) {
            portraitPiPManager.updateIndicatorFromMainView(mPortraitTranslateX, mPortraitTranslateY, mPortraitZoomScale);
        }
    }

    // 更新上次鼠标位置
    mLastMouseNdcX = mouseNdcX;
    mLastMouseNdcY = mouseNdcY;
}
```

3. 在 `resetPortraitZoom()` 中重置状态：
```java
private void resetPortraitZoom() {
    mPortraitZoomScale = 1.0f;
    mPortraitTranslateX = 0f;
    mPortraitTranslateY = 0f;
    mLastMouseNdcX = 0f;
    mLastMouseNdcY = 0f;
    mHasLastMousePosition = false;
    if (activity != null && activity.mCameraHelper != null) {
        activity.mCameraHelper.setZoomScale(1.0f, 0f, 0f);
    }
}
```

4. 在 `adjustZoom()` 中，当缩放级别变化时重置鼠标位置记录：
```java
private void adjustZoom(float zoomFactor) {
    // ... 现有代码 ...
    
    // 缩放级别变化时，重置鼠标位置记录，避免跳跃
    if (portraitPiPManager != null && Math.abs(newScale - mPortraitZoomScale) > 0.01f) {
        mHasLastMousePosition = false;
    }
    
    // ... 继续现有代码 ...
}
```

**参数调优建议**：
- `PAN_SENSITIVITY = 0.3f`：视角移动速度是鼠标移动速度的 30%（较慢，精确控制）
- `PAN_SENSITIVITY = 0.5f`：视角移动速度是鼠标移动速度的 50%（中等）
- `PAN_SENSITIVITY = 0.7f`：视角移动速度是鼠标移动速度的 70%（较快）

**灵敏度计算示例**：
假设 `PAN_SENSITIVITY = 0.5f`，鼠标从视口中心移动到右边缘（移动距离 = 视口宽度的一半）：
- 鼠标 NDC 增量：`deltaMouseNdcX = 1.0`
- 视角平移增量：`deltaTranslateX = -1.0 * 0.5 = -0.5`
- 结果：视角移动了最大平移范围的一半（因为 `maxTranslate = scale - 1`）

这意味着鼠标需要移动**两个视口宽度**才能将视角移到最边缘，比当前逻辑慢了一倍，控制更精确。

### 方案 2：调整显示范围（问题 1）

**可选方案 A：降低最大缩放倍数**

修改 `adjustZoom()` 方法：
```java
// 将 MAX_SCALE 从 3.0 降低到 2.0 或 2.5
float MAX_SCALE = 2.5f;  // 原来是 3.0f
```

**优点**：
- 简单直接
- 放大后能看到更多区域

**缺点**：
- 牺牲了放大倍数的上限

---

**可选方案 B：调整平移范围限制**

当前公式 `maxTranslate = scale - 1` 是正确的（保证纹理不超出视口）。但如果用户希望看到更多区域，可以考虑：

1. **放宽平移限制**：允许纹理部分超出视口（但边缘会留黑）
2. **或者接受当前的限制**：这是正确的数学行为

**建议**：先实施方案 1（修复视角移动速度），然后让用户测试，根据反馈再决定是否调整显示范围。

## 实施步骤

### 第一阶段：修复视角移动速度（问题 2）

1. 修改 `CustomTouchListener.java`：
   - 添加 `mLastMouseNdcX/Y`、`mHasLastMousePosition`、`PAN_SENSITIVITY` 字段
   - 重写 `autoPanToKeepMouseCentered()` 方法
   - 修改 `resetPortraitZoom()` 方法

2. 测试验证：
   - 放大后移动鼠标，视角应该平滑跟随
   - 鼠标移动速度快，视角移动速度也快
   - 鼠标移动速度慢，视角移动速度也慢
   - 可以精确控制视角位置

3. 调优 `PAN_SENSITIVITY` 参数：
   - 从 `0.5f` 开始测试
   - 根据用户反馈调整（范围 0.1-1.0）

### 第二阶段：评估显示范围问题（问题 1）

1. 在第一阶段完成后，让用户测试
2. 如果用户仍然觉得显示范围太小：
   - 尝试降低 `MAX_SCALE`（从 3.0 降到 2.5 或 2.0）
   - 或者接受当前的数学行为（这是正确的放大效果）

## 后续优化建议

1. **添加平滑过渡**：当前视角移动是即时的，可以添加插值动画让移动更平滑
2. **支持惯性滚动**：快速移动鼠标后松手，视角可以继续滑动一段距离
3. **双击复位**：双击视频区域可以快速将视角复位到中心
4. **边缘提示**：当视角移到最边缘时，可以显示一个视觉提示（如边框高亮）

## 参考资料

- OpenGL MVP 矩阵变换：`gl_Position = T * S * R * M * aPosition`
- NDC 坐标空间：`[-1, 1]`，`(-1, -1)` 是左下角，`(1, 1)` 是右上角
- 鼠标增量计算：`delta = current - last`（不是绝对位置）
- 灵敏度参数：控制视角移动速度相对于鼠标移动速度的比例
