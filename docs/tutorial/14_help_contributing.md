# Tutorial 14 — Getting Help and Contributing

**Level:** All levels · **Prerequisites:** Any · **Estimated time:** 5 minutes

You've mastered the app. Here's how to get help when you need it and how to give back.

---

## 14.1 Getting Help

If you run into an issue:

1. **Read the troubleshooting guide** ([Tutorial 10](./10_troubleshooting.md)) — most problems have a known fix
2. **Check the README** — the project's [README.md](../README.md) has setup info and known issues
3. **Visit openterface.com** — the official site has documentation and support
4. **Open a GitHub Issue** — describe your problem, include logs if possible

When opening an issue, include:

- Your Android device model and version
- The app version (found in settings → About)
- What you were doing when the problem occurred
- Any relevant log output

---

## 14.2 Contributing to the App

The Openterface app is open source and welcomes contributions. Here are ways to help:

### Report Bugs

Found something that doesn't work right? Open an issue on GitHub with a description of the problem and how to reproduce it.

### Suggest Features

Have an idea for a new feature? Open a feature request or start a discussion.

### Improve Documentation

If you found a section confusing, others will too. You can:

- Fix unclear explanations in the docs
- Add screenshots or diagrams where helpful
- Translate documentation into your language

### Contribute Code

Browse the [GitHub issues](https://github.com/TechxArtisanStudio/Openterface_Android/issues) for tasks labeled as "good first issue" or "help wanted."

### Mentor Others

If you've mastered the app, help a new contributor get started. Walk them through the setup and first connection — it takes just a few minutes and makes a big difference.

---

## 14.3 Tips for Beginners

- Work through the tutorials in order — each builds on the previous
- If you get stuck, move on and come back later
- Don't be afraid to open an issue with questions — the maintainers are welcoming
- The best way to learn is to try something, break it, and fix it

## 14.4 Tips for Experts

- Focus on the advanced features ([Tutorial 13](./13_advanced.md)) for the highest-impact contributions
- Pay attention to things that feel clunky or slow — those are the best improvement opportunities
- Write a brief description of your approach before starting a big change
- Test on a real device whenever possible
