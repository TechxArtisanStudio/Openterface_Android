# Tutorial 11 — Updates and Versions

**Level:** Beginner · **Prerequisites:** Tutorial 2 · **Estimated time:** 5 minutes

Learn how the app is versioned and how to keep up to date.

---

## 11.1 Understanding Version Numbers

The app uses **Semantic Versioning**: `major.minor.patch`

| Part | Meaning | Example |
|---|---|---|
| **Major** | Big changes that may not be backwards-compatible | 1.0.0 → 2.0.0 |
| **Minor** | New features, backwards-compatible | 1.2.0 → 1.3.0 |
| **Patch** | Bug fixes only, backwards-compatible | 1.2.2 → 1.2.3 |

For example, version **1.2.3** means: major version 1, minor version 2, patch 3.

---

## 11.2 Checking Your Version

Open the settings panel and tap **About Device**. This shows the current app version and your Android version.

---

## 11.3 Getting Updates

Depending on how you installed the app:

- **Play Store:** Updates are delivered automatically
- **GitHub Releases:** Visit the [Releases page](https://github.com/TechxArtisanStudio/Openterface_Android/releases) and download the latest APK

---

## 11.4 Release Notes

Each release on GitHub includes a list of changes:

- New features
- Bug fixes
- Known issues

Read the release notes before updating to understand what's changed.

---

## 11.5 Should You Update?

- **Patch releases** (e.g., 1.2.2 → 1.2.3) — always safe, just bug fixes
- **Minor releases** (e.g., 1.2.0 → 1.3.0) — new features, generally safe
- **Major releases** (e.g., 1.x → 2.0.0) — read the release notes carefully for breaking changes
